# 002: 権限制御(IDOR)に関する横断的セキュリティレビュー

- 種別: セキュリティレビュー
- 優先度: 高
- ステータス: 未着手(推奨事項)

## 背景・現状

- バグ管理表(全28件、全て修正完了)のうち6件(No.8, 10, 13, 21, 24, 28)は、URLパラメータ改ざんにより本来アクセス権限のないリソース(他家族の子供のクエスト編集/削除/承認/却下、他人の支出上限却下、自分自身の管理者アカウント削除等)を操作できてしまうIDOR(Insecure Direct Object Reference)系の不具合だった。
- 個別の不具合は再テストで修正確認済みだが、同種の「所有者チェック漏れ」というパターンが繰り返し検出されていたことから、システムテスト設計書がカバーしていない他のPOST/更新系エンドポイントにも同様の考慮漏れが残っている可能性がある。

## Todo

- [ ] 1. 全ControllerのPOST/更新系エンドポイント(クエスト・収支記録・支出上限・キャラクター・アカウント関連)を棚卸しし、パスパラメータで指定されるリソースIDに対して「ログインユーザーが所有者/家族関係にあるか」のチェックが各Serviceメソッドで行われているか確認する。
- [ ] 2. チェック漏れが見つかった箇所には、修正済みの箇所(例: QuestService)と同様の所有者チェックを追加する。
- [ ] 3. 可能であれば、他ユーザーIDを付与したリクエストを送る権限バイパステストをシステムテストに追加する。

## 対象箇所

- `src/main/java/com/example/moneyquest/app/presentation/controller/` 配下 全Controller
- `src/main/java/com/example/moneyquest/app/domain/service/` 配下 全Service
- 004_テスト/マネークエスト_バグ管理表.xlsx (No.8, 10, 13, 21, 24, 28)

## 備考

ドキュメント上は全バグ修正完了・全テストOKで「完了」扱いだが、これは既知の不具合の再発防止であり、未知の類似不具合の有無を保証するものではないため、追加確認を推奨する。
