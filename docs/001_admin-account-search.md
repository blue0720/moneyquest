# 001: 管理者アカウント一覧に検索機能が実装されていない疑い

- 種別: 実装漏れ疑い
- 優先度: 中
- ステータス: 完了

## 背景・現状

- 機能一覧表(機能一覧_マネークエスト_v1.1.xlsx)を再確認したところ、管理者アカウント一覧タブの検索は **FR-04-4「管理者アカウント一覧表示」**(「ここから管理者アカウントの検索・削除をすることも可能」)が正しい根拠だった(起票時に参照したFR-03-5は保護者アカウントタブ側の検索機能で、管理者タブとは無関係。ticket作成時の参照間違い)。
- 実装(admin-home.html)を確認したところ、検索ボックス(`#searchInput` / `filterParents()`)は『保護者アカウント一覧』タブにのみ存在し、『管理者アカウント一覧』タブには検索UIが無いことを実機(コード+実際にアプリを起動してのHTTPレスポンス)で確認した。
- 検索方式について: FR-03-5(保護者側)の仕様文言は「DBから部分一致検索する」だが、実装済みの保護者タブは実際にはクライアントサイドJSフィルタで動いており、これは既にテスト完了・本番相当の挙動として確定している。今回追加する管理者タブの検索も、コードベースの一貫性を優先し同じクライアントサイドJSフィルタ方式で実装した(サーバー側検索への統一は別途の判断が必要な仕様変更のため、今回のスコープ外とした)。

## 実施内容

- `src/main/resources/templates/admin-home.html` の管理者管理タブに、保護者管理タブと同じ構造の検索バー(`#adminSearchInput`)を追加。
- 各管理者カードに `th:data-name` / `th:data-email` 属性を追加(`filterParents()`と同じ仕組みでフィルタできるように)。
- 検索結果が0件の場合に表示する `#adminEmpty` の空状態メッセージを追加(保護者タブの `#parentEmpty` と同仕様)。
- `filterAdmins()` 関数を追加(`filterParents()`と同ロジック、対象を`#adminCardList`に変更)。
