# 005: 表示確認・リンク動作確認・主要ブラウザ動作確認 テストケース

- 種別: テスト設計
- 優先度: 中
- ステータス: 実施済み(Chromeのみ。Edge/Firefox/Safariは未実施、下記参照)

## 目的・背景

- 全画面(タブ含む)を対象に、①レイアウト崩れがないかの「表示確認」、②タブ・リンク・ボタンの遷移先が正しいかの「リンク動作確認」、③主要ブラウザ間で表示・動作に差異がないかの「クロスブラウザ確認」の3観点でテストケースを整理する。
- 対象画面は `TransitionTargetPageNameKeyword` で定義されているルート、および `parent-home.html` / `child-home.html` / `admin-home.html` 内の `activeTab` で切り替わる各タブとする(詳細は [CLAUDE.md](../CLAUDE.md) の「テンプレートは『画面ごと』ではなく『ロールごと』」を参照)。
- 本テストは表示・遷移の確認が目的であり、業務ロジック(バリデーション・権限制御等)の正誤は対象外(権限制御は [002_authorization-security-review.md](./002_authorization-security-review.md) を参照)。

## テスト対象ブラウザ

| No | ブラウザ | 想定バージョン | 備考 |
|---|---|---|---|
| B1 | Google Chrome | 最新安定版 | 開発・デバッグの主ブラウザ |
| B2 | Microsoft Edge (Chromium) | 最新安定版 | Windows標準ブラウザ |
| B3 | Mozilla Firefox | 最新安定版 | |
| B4 | Safari | 最新安定版(macOS/iOS) | 環境がない場合は対象外とし、その旨を備考に明記する |

## 実施手順(共通)

1. `application.properties` の `spring.profiles.active` を `prod` にした状態(認証あり)で実施する。
2. MySQL80 サービスを起動し、`./mvnw spring-boot:run` でアプリを起動する。
3. 保護者・子供・管理者それぞれのテストアカウントでログインし、各タブを一巡する。
4. ウィンドウ幅は PC 標準サイズ(1366×768 以上)を基本とし、崩れが疑われる場合はブラウザの開発者ツールでウィンドウ幅を縮小して再確認する。
5. 結果は各表の「結果」列に `OK` / `NG` を記入し、`NG` の場合は「備考」に事象と再現条件を記載する。NGはバグ管理表へ起票し、バグ番号を備考に追記する。
